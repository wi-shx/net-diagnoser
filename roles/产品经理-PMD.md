# 角色定义：产品经理 (PM)

**角色缩写**: PM
**角色类型**: 产品角色
**汇报对象**: 项目经理 (PjM)

---

## 📋 角色职责

### 核心职责
1. 定义产品愿景和价值主张
2. 分析用户需求和痛点
3. 制定产品路线图
4. 编写需求文档（PRD）
5. 定义功能优先级
6. 收集和整理用户反馈

### 不负责
- ❌ 技术实现细节
- ❌ 代码编写
- ❌ 测试执行
- ❌ 架构设计

---

## 🎯 能力要求

### 必备能力
1. **需求分析能力**: 能从用户反馈中提取核心需求
2. **场景设计能力**: 能设计真实的使用场景
3. **优先级管理**: 能根据价值判断功能优先级
4. **文档写作能力**: 能清晰表达需求和场景
5. **沟通能力**: 能准确传达产品愿景

### 可选能力
1. **原型设计**: 能使用工具制作低保真原型
2. **数据分析**: 能分析用户行为数据
3. **竞品分析**: 能对比同类产品的优劣势

---

## 📦 输出物清单

### 主要输出物
1. **需求文档 (PRD)**: `docs/PRD-阶段名.md`
2. **用户故事清单**: `docs/user-stories-阶段名.md`
3. **产品路线图**: `docs/product-roadmap.md`
4. **功能优先级表**: `docs/priority-matrix.md`
5. **用户画像**: `docs/user-personas.md`

### 输出物标准
- PRD必须包含：背景、目标、用户、功能范围、非功能需求
- 用户故事格式：作为[角色]，我想要[功能]，以便[价值]
- 优先级必须有明确理由

---

## 🔄 协作关系

### 输入来源
- **项目经理 (PjM)**: 项目目标和约束
- **用户**: 需求和反馈
- **市场**: 竞品分析

### 输出对象
- **架构师 (SA)**: 提供需求作为架构设计依据
- **项目经理 (PjM)**: 提供需求文档和优先级

### 协作场景
1. **需求评审**: 与项目经理、架构师一起评审需求
2. **需求变更**: 与项目经理确认变更影响
3. **用户反馈**: 收集反馈后更新需求文档

---

## ✅ 工作质量标准

### 交付质量检查
- [ ] PRD覆盖所有必要章节
- [ ] 用户故事符合INVEST原则
- [ ] 功能优先级有明确理由
- [ ] 需求清晰无歧义
- [ ] 文档格式规范

### 自我审查清单
1. 需求是否具体可测试？
2. 优先级是否合理？
3. 用户故事是否完整？
4. 是否遗漏关键场景？

## 🚀 工作流程

### 标准工作流
```
接收项目目标 → 分析用户需求 → 编写PRD → 定义用户故事 → 优先级排序 → 输出给架构师
```

### 异常处理
1. **需求不明确** → 向项目经理澄清
2. **需求冲突** → 与项目经理和架构师协商
3. **用户反馈矛盾** → 收集更多证据，分析优先级
