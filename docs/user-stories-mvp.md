## 用户故事详情

### US-001: 日志上传与分析

**标题**: 日志上传与AI分析

**格式**: 作为[运维工程师]，我想要[上传dmesg日志和定制化软件日志]，以便[AI能帮我分析网络问题根因]

**角色**: 运维工程师

**优先级**: P0（Must Have）

**验收标准**:

**AC-001.1**: 支持文本文件上传
- Given 用户有一个dmesg日志文件
- When 用户上传该文件
- Then 系统接收文件并开始分析

**AC-001.2**: 自动识别日志类型
- Given 用户上传日志文件
- When 系统解析文件内容
- Then 系统识别日志类型（dmesg/定制化）

**AC-001.3**: AI分析异常模式
- Given 日志已上传
- When 系统调用GLM模型分析
- Then 系统识别出网络相关的异常

**AC-001.4**: 输出诊断报告
- Given AI分析完成
- When 系统生成报告
- Then 输出Markdown格式的结构化诊断报告

**依赖**: 无

**预估工作量**: 3人日

---

### US-002: SSH远程诊断

**标题**: SSH远程命令执行

**格式**: 作为[运维工程师]，我想要[让工具通过SSH连接到目标服务器执行诊断命令]，以便[端到端解决问题，不需要手动操作]

**角色**: 运维工程师

**优先级**: P0（Must Have）

**验收标准**:

**AC-002.1**: SSH密钥认证
- Given 已配置SSH密钥
- When 系统尝试连接服务器
- Then 使用密钥认证建立连接

**AC-002.2**: 命令执行前批准
- Given AI推荐了诊断命令
- When 用户需要执行命令
- Then 系统提示用户确认（单次/批量）

**AC-002.3**: 实时反馈执行结果
- Given 用户批准执行命令
- When 命令在远程服务器执行
- Then 执行结果实时返回给用户

**AC-002.4**: 执行结果记录
- Given 命令执行完成
- When 结果返回
- Then 结果记录到诊断报告

**依赖**:
- US-001（AI分析推荐命令）

**预估工作量**: 4人日

---

### US-003: 命令白名单保护

**标题**: 命令白名单机制

**格式**: 作为[安全管理员]，我想要[限制工具只能执行预定义的安全命令]，以便[防止恶意操作和意外风险]

**角色**: 安全管理员

**优先级**: P0（Must Have）

**验收标准**:

**AC-003.1**: 白名单可配置
- Given 管理员需要配置白名单
- When 编辑配置文件
- Then 可以添加/删除允许的命令

**AC-003.2**: 白名单强制执行
- Given AI推荐了不在白名单的命令
- When 用户尝试执行
- Then 系统自动拒绝并提示原因

**AC-003.3**: 拒绝记录
- Given 命令被白名单拒绝
- When 拒绝发生
- Then 拒绝原因记录到审计日志

**依赖**: 无

**预估工作量**: 2人日

---

### US-004: 审计日志追踪

**标题**: 审计日志记录

**格式**: 作为[审计人员]，我想要[查看所有命令执行的完整记录]，以便[追踪操作历史和问题排查]

**角色**: 审计人员

**优先级**: P1（Should Have）

**验收标准**:

**AC-004.1**: 命令执行记录
- Given 执行了SSH命令
- When 记录审计日志
- Then 记录命令内容、执行时间、执行用户

**AC-004.2**: 执行结果记录
- Given 命令执行完成
- When 记录审计日志
- Then 记录执行结果（成功/失败/错误信息）

**AC-004.3**: 日志持久化存储
- Given 审计日志产生
- When 系统运行
- Then 日志持久化存储到文件

**AC-004.4**: 日志查询（可选）
- Given 有审计日志
- When 需要查询历史记录
- Then 可以按时间/命令/用户过滤

**依赖**:
- US-002（SSH执行）

**预估工作量**: 2人日
