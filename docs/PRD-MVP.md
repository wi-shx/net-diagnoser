## 🎯 产品定位

### 一句话描述
一个AI驱动的网络故障诊断工具，通过日志上传和智能分析，快速定位服务器网络问题。

### 核心价值
1. **效率提升**：将网络故障排查时间从小时级降到分钟级
2. **降低门槛**：让不熟悉网络工具的开发人员也能自助排查
3. **知识沉淀**：积累故障案例，形成组织资产

### 与竞品差异
- **轻量级**：本地部署，无需复杂基础设施
- **AI驱动**：智能分析日志，而不仅仅是展示
- **可扩展**：支持从AI分析演进到Agent自主排查

## 🔧 核心功能详细需求

### 1. 日志上传

**需求描述**：
用户通过CLI上传本地的日志文件，工具接收并解析。

**输入**：
- 日志文件路径（绝对路径或相对路径）
- 文件格式（可选，自动检测）

**输出**：
- 上传成功/失败提示
- 文件基本信息（大小、行数、检测到的格式）

**验收标准**：
- [ ] 支持.log和.txt格式文件
- [ ] 单个文件最大支持100MB
- [ ] 文件不存在时给出友好提示
- [ ] 上传进度显示（可选）

**示例命令**：
```bash
netdiagnoser analyze --log /var/log/nginx/error.log
```

---

### 2. 日志解析

**需求描述**：
解析日志文件，提取关键信息（时间戳、错误级别、IP地址、端口号、状态码等）。

**支持格式**：

#### Nginx访问日志
```
127.0.0.1 - - [15/Feb/2026:10:00:00 +0800] "GET /api HTTP/1.1" 500 123
```
提取：IP、时间、请求方法、URL、状态码、响应大小

#### Haproxy日志
```
Feb 15 10:00:00 localhost haproxy[1234]: 127.0.0.1:12345 [15/Feb/2026:10:00:00.123] backend1 server1 0/0/0/1 500 0 - ----
```
提取：时间、IP、backend名称、server名称、状态码

#### Syslog
```
Feb 15 10:00:00 server kernel: [12345.678] TCP: time wait bucket table overflow
```
提取：时间、进程、级别、消息

**输出**：
- 解析后的结构化数据（JSON格式）
- 错误日志统计（按错误级别、类型分组）

**验收标准**：
- [ ] 至少支持3种日志格式
- [ ] 无法识别的格式给出提示
- [ ] 解析失败行单独记录
- [ ] 提取关键信息准确率≥95%

---

### 3. AI分析

**需求描述**：
使用GLM-4.7/5.0分析解析后的日志，识别问题模式和可能原因。

**分析维度**：
1. **问题类型**：连接超时、DNS解析失败、端口不可达、高延迟、服务异常
2. **错误模式**：错误频率、错误分布、相关性分析
3. **可能原因**：网络配置问题、防火墙限制、服务故障、DNS问题、资源不足
4. **风险等级**：P0（服务完全不可用）、P1（部分功能受影响）、P2（偶发问题）

**AI Prompt要求**：
```
你是一个网络诊断专家，请分析以下日志：

[日志内容]

请输出：
1. 问题类型
2. 可能原因（Top 3）
3. 风险等级（P0/P1/P2）
4. 建议的排查命令
```

**输出**：
- JSON格式的分析结果
- 包含问题类型、可能原因、风险等级、建议命令

**验收标准**：
- [ ] 调用GLM API成功
- [ ] 分析结果结构化（JSON格式）
- [ ] 问题识别准确率≥80%（人工抽样评估）
- [ ] API调用超时重试机制

---

### 4. 诊断报告

**需求描述**：
生成结构化的Markdown格式诊断报告，包含日志概览、AI分析结果和建议命令。

**报告结构**：
```markdown
# 网络诊断报告

## 文件信息
- 文件名：error.log
- 文件大小：1.2 MB
- 日志格式：nginx access log
- 分析时间：2026-02-15 10:00:00

## 日志概览
- 总日志行数：10000
- 错误行数：500
- 错误率：5%

## AI分析结果

### 问题类型
连接超时

### 可能原因
1. 服务器防火墙限制
2. 目标服务未启动
3. 网络路由问题

### 风险等级
P0（服务完全不可用）

## 建议排查命令

```bash
# 检查防火墙规则
sudo iptables -L -n

# 检查服务状态
sudo systemctl status nginx

# 测试网络连通性
ping target-server.com
```

## 下一步
1. 执行上述命令收集更多信息
2. 根据结果调整排查方向
3. 如需进一步分析，可提供更多日志
```

**输出**：
- Markdown格式报告文件
- 同时打印到终端

**验收标准**：
- [ ] 报告结构清晰，易读
- [ ] 包含文件信息、日志概览、分析结果、建议命令
- [ ] Markdown格式正确
- [ ] 报告自动保存到reports/目录

---

### 5. 命令建议

**需求描述**：
根据AI分析结果，提供可执行的Linux命令，帮助用户进一步排查。

**命令分类**：
- **网络连通性**：ping, traceroute, mtr
- **端口检查**：telnet, nc, ss, netstat
- **DNS解析**：dig, nslookup
- **服务状态**：systemctl, service
- **防火墙检查**：iptables, firewalld, ufw

**命令格式**：
```bash
# 命令说明
命令本身

# 示例
# 检查端口80是否开放
sudo ss -tlnp | grep :80
```

**验收标准**：
- [ ] 至少提供3类命令（网络、端口、DNS）
- [ ] 命令格式包含说明和示例
- [ ] 命令适用于Linux系统
- [ ] 避免危险命令（如rm、重启服务）

---

### 6. CLI界面

**需求描述**：
提供友好的命令行交互界面，支持多种使用方式。

**核心命令**：
```bash
# 分析单个日志文件
netdiagnoser analyze --log <file>

# 分析多个日志文件
netdiagnoser analyze --log <file1> --log <file2>

# 指定日志格式
netdiagnoser analyze --log <file> --format nginx

# 指定AI模型
netdiagnoser analyze --log <file> --model glm-5.0

# 查看版本信息
netdiagnoser --version

# 查看帮助
netdiagnoser --help
```

**交互体验**：
- 彩色输出（使用Rich库）
- 进度条显示（日志解析、AI调用）
- 错误提示友好

**验收标准**：
- [ ] 核心命令可用
- [ ] 参数验证完善
- [ ] 帮助文档清晰
- [ ] 错误提示友好

## 🚫 非功能性需求

### 安全性
- 日志文件不上传到云端（可选，AI API除外）
- 命令建议不包含危险操作
- 敏感信息（密码、token）不显示在报告中

### 可靠性
- 日志解析失败时继续处理有效行
- AI API调用失败时给出明确提示
- 支持断点续传（日志解析中断后可继续）

### 可维护性
- 代码结构清晰，模块化设计
- 配置文件支持（.env或config.yaml）
- 详细的日志记录（debug级别）
